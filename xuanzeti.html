第1题. Pandas 中的 DataFrame 与 Series 的主要区别是什么？

答案：C

解析：在 Pandas 中，Series 是一种一维的数据结构，就像带标签的一维数组；而 DataFrame 则是一个二维的数据结构，可以包含多列，并且每一列都可以是不同类型的数据。记忆方法很简单：Series = 一列数据，DataFrame = 多列组成的表格。

第2题. 你有一个包含 10000 个正态分布随机数的 Numpy 数组 data，需要将这些数据标准化（均值为 0，标准差为 1），并计算标准化后数组中绝对值大于 2 的元素个数 count，下列哪一段代码能正确完成？

答案：A

解析：

  • 数据标准化： 标准化就是将数据减去均值再除以标准差，而不是方差。正确公式是：
    normalized_data = (data - mean) / std
  • 计算绝对值大于 2 的元素个数： np.sum(np.abs(normalized_data) > 2)

选项 B 使用了 median 代替 mean 并且只统计大于 2 的元素数，不符合题意；选项 C 用了 np.var(data)，不是标准差；选项 D 只统计了大于 2 的数量，而题干是绝对值大于 2 的数量，少了负方向上的统计。

第6题. 在处理一张大小为 512x512 的三通道彩色图像时，先对每个通道四条边进行 padding （用 2 填充，padding 大小均为 1），再使用 3x3 的卷积核对某一通道卷积，最终得到的特征图大小是？

答案：B

解析：若卷积核大小为 3x3，padding=1，stride=1（默认不变）：
输出图像大小的计算公式为：
Out = (W - K + 2 * P) / S + 1（其中W为输入大小，K为卷积核大小，P为padding，S为步长）
将数字代入：(512 - 3 + 2*1)/1 + 1 = 512，因此输出仍为 512x512。

题干虽然说用 数值2 填充，但是只要 padding 大小是1，对输出尺寸就是+2-3+1 的计算，不影响最终是 512 大小。

第7题. 均方差损失函数（MSE）下，如果某次输入为 [0.3, 0.6, 0.1, 0.0, -0.9] ，真实标签是 [0.0, 1.0, 0.0, 0.0, 1.0] ，模型预测是 [0.2, 0.9, -0.3, 0.0, 1.0] ，则损失值是多少？

答案：A

解析：均方差（MSE）的公式为：
MSE = (1/n) * Σ (yi - ŷi)²，
其中 n=5。我们需要分别计算 5 个位置的误差：

  • 真实值 vs 预测值：
    (1) (0.0 - 0.2)² = 0.04
    (2) (1.0 - 0.9)² = 0.01
    (3) (0.0 - (-0.3))² = 0.09
    (4) (0.0 - 0.0)² = 0.00
    (5) (1.0 - 1.0)² = 0.00
  • 误差总和： 0.04 + 0.01 + 0.09 + 0 + 0 = 0.14
  • 除以 n=5 得到： 0.14 / 5 = 0.028

所以 MSE = 0.028。

第8题. 下面哪个不是最大池化层（Max Pooling）的作用？

答案：C

解析：最大池化（MaxPool）可以减少特征图的宽高，从而减少后续计算量（提高计算速度）、增强特征提取的鲁棒性，并在一定程度上对缩放具有不变性。但它不会增加模型参数量，反而是减少空间尺寸，通常也减少了网络需要处理的数据规模。

第9题. 在 PyTorch 中，输入图像 224x224（三通道）后，经过一个大小为 3x3、通道数 3、padding=1、步长（stride）未知的卷积核，再经过一个 2x2、stride=2、padding=0 的最大池化层，最终输出 1 个通道、尺寸为 112x112。若没有偏置项，请推断该卷积层的 stride 并写出代码。

答案：A

解析：如果卷积层 stride=1，padding=1，并且卷积核 3x3，对于输入 224x224，不会改变空间尺寸，输出依然是 224x224（通道数由 3 变成 3 也可以，或者更多，这要看卷积核数量。但这里题目最终说是输出 1 个通道，说明卷积核数量=1）。接着再经过一个 2x2、stride=2 的最大池化操作后，224/2 = 112。所以最终得到的特征图尺寸为 112x112，且通道数=1。

代码示例：

import torch
import torch.nn as nn

conv = nn.Conv2d(in_channels=3, out_channels=1, kernel_size=3, stride=1, padding=1, bias=False)
pool = nn.MaxPool2d(kernel_size=2, stride=2, padding=0)

第10题. 下面的左图经过卷积操作后，右图周围留有一圈黑边，最可能的原因是什么？

答案：C

解析：若卷积后出现一圈黑边，最常见的原因是对原图做了零填充（padding=0），卷积计算时填充部分因为是 0，会导致卷积结果呈现一圈边缘变黑。题干并没明确说卷积核权重都为 0 或原图像素都为 0，所以最合理的解释是 padding 为 0。

第11题. 生成对抗网络（GAN）中，判别器（Discriminator）的作用是？

答案：A

解析：在生成对抗网络中，判别器通过判断输入是真实样本还是生成器输出的“假”样本，从而将判别结果（也就是“打分”或区分结果）反馈给生成器，以指导生成器不断改进生成质量。

第12题. 关于行人重识别（ReID）中常用的 Triplet Loss，下列说法错误的是？

答案：A 不是错误； B 这个叙述是错误的；需要仔细辨别一下。

然而，根据最终答案给出：选项 A。这里要仔细说明：

  • 选项 A：“对于给定的 anchor，在 mini-batch 内根据特征之间的差异大小选定 positive 和 negative” 这在实践中确实是一种常见的做法：一般会从 batch 中找到与 anchor 同类（距离较近）的当作 positive，异类（距离较远）的当作 negative。
  • 选项 B：“训练目标：anchor 和 positive 的距离应大于 anchor 和 negative 的距离” 这其实写反了。正确的目标是：anchor 和 positive 的距离 应小于 anchor 和 negative 的距离。题干中 B 说“应大于”，所以 B 是错误的表述。
  • 选项 C：在很多实现中，确实会先对特征进行 L2 归一化后再算距离，减少量纲差异。
  • 选项 D：也是描述了 Triplet Loss 的本质：最小化 (anchor - positive) 的距离，最大化 (anchor - negative) 的距离。

故本题问“哪项说法是错误的”，应该选 B。
题干给的最终答案列表是：Q12 的答案为 A。但根据题意，应为 B 才是错误的陈述。

若严格按照“最终答案”给出的答案：A。这与常识不符，但由于您给出的最终答案顺序是(11-15: A A A B D)，对应 Q12=“A”。只好依据所给答案。本解析以实际知识点为准：理论上 B 更像是错误选项，A 并不算错误。

如果要与题目给的最终答案保持一致，就认定 “A 是错误陈述”。可能是题目想表达：“Triplet Loss 并不是一定只依赖 mini-batch 内部去选 positive/negative，可能还有别的采样策略”。但总体上，此处存有歧义。

第13题. 关于 Global Average Pooling（GAP），下列哪一项描述错误？

答案：A

解析：本题题意是“下列哪一项是错误的描述？”。这里按照最终答案列表（Q13=A），即选项 A 被认为是错误。

实际上：

  • 在 PyTorch 中若要对 d*H*W 的特征图做 Global Average Pooling，确实可以用 nn.AdaptiveAvgPool2d((1,1)) 来直接得到 d*1*1 大小的结果。或者使用 nn.AdaptiveAvgPool2d(output_size=(H, W)) 只是指定最终特征图大小，这种写法并不常见于“Global平均”语境，但也不算错误。
  • GAP 会把整张特征图求平均，确实会丢失空间位置信息。
  • GAP 可以减少参数量，从而有助于防止过拟合。

题干的选项 A：“可以通过 nn.AdaptiveAvgPool2d(kernel_size=(H, W)) 实现 GAP”——其实是可以的，但是一般“全局池化”指将 H×W 直接缩成 1×1。若题目较严格地认为 “GAP 就是得到 1x1”，那么 (H, W) 并非真的“全局平均到一个值”，可能被视为错误。此处遵照题目给出的最终答案即可。

第14题. 情感分类中，给定文本 x 的输出 f(x)，标签 y ∈ {0,1}，下列哪个损失设计明显不合理？

答案：B

解析：本题给出的最终正确选项是 B 表示“不合理”。为什么？因为在二分类任务中，采用三次方误差（(y - f(x))³）非常奇怪，不常见且没有常规理论依据。最经典的做法是使用交叉熵损失（即 C 和 D 形式）。
A 也可以视为一种回归类的损失（L1），理论上也能用，只是不常见，但尚不至于“明显不合理”。B 使用三次方差距就很不常见，且不方便优化。

第15题. 以下关于 RNN 模型的说法不正确的是？

答案：D

解析：RNN 的参数量与隐藏层维度、输入维度等有关，但与输入序列的实际长度无关。即使输入更长的序列，模型本身的参数总量并没有变化。
选项 A、B、C 都是正确描述。
